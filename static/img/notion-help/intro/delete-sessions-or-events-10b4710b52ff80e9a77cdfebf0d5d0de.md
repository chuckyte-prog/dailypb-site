# Delete Sessions or Events

# Deleting a Single Session

1. Click on the Session you want to delete
2. Scroll to the bottom and click on “Cancel / Delete” in the Tools menu.
3. Click on “Delete and Remove Session.”
4. Confirmed by clicking “Yes, Delete” to confirm you want to delete this Session. 

# Deleting Recurring Sessions

1. Click on the Session you want to delete
2. Scroll to the bottom and click on “Cancel / Delete” in the Tools menu.
3. Click on “Delete and Remove Session.”
4. When the Session you are deleting is part of a reoccurring Event, you will now have the option to select “This and following sessions.” Selecting this will delete all future sessions. 
    
    ![image.png](Delete%20Sessions%20or%20Events/image.png)
    
5.  Whichever days this reoccurring Event falls on, you will have the ability to selectively delete them. 
    
    ![image.png](Delete%20Sessions%20or%20Events/image%201.png)
    
6.  When you are ready click “Yes, Delete”

# Considerations

## Signups

When a Session is deleted, you can only view the Signups associated with that Session via the [host](https://www.notion.so/DailyPB-79ca133edc3b463f8ca2d75932801e59?pvs=21) page and clicking the sign-ups tab. 

Signups will not be notified of the deletion, so it's best to message them beforehand.

## Transactions

**Automatic Payment** Transactions for Signups in a deleted Session will remain accessible in the [Financial Dashboard](https://www.notion.so/DailyPB-79ca133edc3b463f8ca2d75932801e59?pvs=21). Refunds can be issued from there, even though the Session will no longer be accessible.
**Manual Payment** Transactions will be deleted from DailyPB if the Session is deleted before it starts, and the Main Host will not be responsible for any DailyPB fees for those Signups. However, Transactions from past Sessions will remain, even if the Session is deleted.
