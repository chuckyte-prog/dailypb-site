# Notification Settings

In your profile you’ll see some options for Host Notification Preferences.

## **Notify me when someone signs up for my session**

By default this is “on” and will send you an email each time there is a new signup with the signup info as well as the total number of players that have signed up to your Session. 

## **Send a notification to players when they’re manually marked as paid**

By default this is *off*. When *on,* it will send an email to a Player  informing them that their payment has been accepted.  This will only send an email if the Event Host in first position has this setting as *on.*  

This notification is useful for hosts that run many Sessions concurrently as players payments can become confusing fast when using alternative payment methods.

<aside>
💡

Using Automatic Payments requires a successful payment before the Player is allowed to Sign up — removing the need for this notification. Also players who click the Venmo link from a mobile device will have the Venmo notes auto-populated with the Session info, which is helpful for tracking payments. 

</aside>
