# Cancelling a Session

<aside>
💡

Before cancelling, if this is a Session with many Signups, consider simply updating the date or time of the Session and notifying Signups. This will leave all your Signups intact. 

</aside>

To cancel a Session, a Host will need to 

1. Find the Session to be cancel, scroll down to Session Tools and click the Cancel / Delete Session button. 
    
    ![image.png](/img/notion-help/cancelling-a-session/image.png)
    
2. Click the button “Show Session as Cancelled.” The Session will now show as cancelled.
    
    ![image.png](/img/notion-help/cancelling-a-session/image-1.png)
    
3. If the host wants to type a reason for the cancellation they can type it in the text box that will appear and then click “Submit Reason for Cancellation.” This reason will be public. 
4. A checkbox will appear to automatically email all players signed up for the session (if applicable). If the host provides a reason for cancellation in step 3 and checks the box, clicking the "Submit Reason for Cancellation" button will send an email to all Attendees, notifying them of the cancellation and the provided reason.

![image.png](/img/notion-help/cancelling-a-session/image-2.png)

## Other things that happen when a Session is Cancelled

The 2 and 24 hour reminders (if applicable) will also not go out.

Pre and Post Templates (if applicable) will not go out.

The waitlist will not auto-bump the the first person in first position on the waitlist if someone cancels there spot in a cancelled Session. 

Join Event and Join Waitlist buttons will not appear.

On the Attendees popup, users will not be able to use the Manually Add Attendee section as it will not appear.
